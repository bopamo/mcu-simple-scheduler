Frequently Asked Questions (FAQ) v0.2.1

Q1: Why another embedded operating system/scheduler?

A1: Because i can :) .. no, kidding. . basically programming has always been a hobby and s passion for me personally. I have been collecting the basic concepts of this light-weight multi-tasking system from my working experiences, and one night "voila!" i just decided to start working on MSS.

Q2: Which scheduling policies does MSS support?

A2: MSS supports cooperative (tasks will be executed as long as it wants) and preemptive (high priority tasks will preempt the lower prioirty tasks) scheduling policy. Refer to MSS_PREEMPTIVE_SCHEDUILING configuration option in mss_cfg.h.

Q3: What is the maximum number of tasks running on MSS?

A3: Although it hasn't been really tested, MSS should be able to run up to 32 tasks. Refer to mss_hal_int.h.

Q4: How big is the memory footprint of MSS?

A4: Basically the MSS_FOOTPRINT project is created for answering this question. For MSP430 Value Line platform and all activated module, it will take around 1 KB of Flash and 120 bytes of RAM.

Q5: Which platforms are supported currently by MSS?

A5: Refer to Overview page.

Q6: How to import CCS (Code Composer Studio) example projects?

A6: When trying to import the example projects, the option "Copy projects into workspace" shall not be checked.
